// spend_protocol

Deterministic Spend Flow

Agents need execution logic, not storytelling. Every transaction follows an explicit, verifiable protocol.

1. request_spend(agent_id, amount, merchant)

2. Policy engine evaluates constraints

3. Response: APPROVED | DENIED | REQUIRE_APPROVAL

4. If approved → issue_virtual_card()

5. Transaction logged with:

- policy_matched

- decision_reason

- risk_score

- timestamp

// quickstart

10-Line Integration

Runnable in minutes. Install SDK, create agent, execute spend.

quickstart.py

# pip install zinc-sdk

from zinc import Zinc, Policy

 

zinc = Zinc(api_key="zk_...")

agent = zinc.create_agent(name="procurement-bot")

 

policy = Policy(spend_limit=1000, merchant=["aws"])

result = agent.request_spend(amount=500, policy=policy)

 

zinc.on("transaction.approved", handler)
