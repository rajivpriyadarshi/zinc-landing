Deterministic Spend Flow

Agents need execution logic, not storytelling. Every transaction follows an explicit, verifiable protocol.

1. request_spend(agent_id, amount, merchant)

2. Policy engine evaluates constraints

3. Response: APPROVED | DENIED | REQUIRE_APPROVAL

4. If approved → issue_virtual_card()

5. Transaction logged with:

- policy_matched

- decision_reason

- risk_score

- timestamp

How It Works

Five steps to autonomous transactions

From policy definition to automatic verification—complete financial autonomy for your agents.

01

Register Agent & Set Policies

Register your AI agent and define comprehensive spend controls—limits per transaction, daily caps, allowed merchants, blocked categories, and velocity controls. Your policies become the guardrails for every future transaction.

Spend Limits Merchant Locks Velocity Controls
// Register agent with policies
await zinc.agents.create({
  name: "procurement-bot",
  policies: {
    perTransaction: 5000,
    perDay: 20000,
    merchants: ["aws", "gcp"]
  }
});
Agent Registered
02

Declare Intent

Before any purchase, your agent declares exactly what it wants to buy, from which merchant, and for how much. This intent becomes the contract that all subsequent actions are verified against.

Pre-authorization Intent Logging
// Declare purchase intent
const intent = await zinc.intents.create({
  agentId: "procurement-bot",
  merchant: "aws.amazon.com",
  amount: 2500,
  purpose: "EC2 instances"
});
Awaiting Policy Check
03

Card Issuance

If the intent passes your policies, a virtual card is automatically created and locked to the declared intent. The card can only be used at the specified merchant for the approved amount.

Virtual Cards Intent-Locked
// Card auto-generated on approval
const card = await zinc.cards.issue({
  intentId: intent.id,
  type: "single_use"
});

// Returns: PAN, CVV, expiry
Card Ready
04

Execute Checkout

Your agent retrieves the card credentials and completes the purchase at the merchant. The transaction is constrained to the pre-approved parameters—any deviation is automatically blocked.

Secure Checkout Auto-Block
// Agent completes purchase
const txn = await agent.checkout({
  card: card.credentials,
  merchant: "aws.amazon.com",
  amount: 2500
});

// status: "completed"
Purchase Complete
05

Auto Verification

Every transaction is automatically verified against the declared intent. Mismatches are flagged, receipts are captured, and a complete audit trail is generated for compliance and reconciliation.

Intent Matching Audit Trail Receipts
// Automatic verification
zinc.verify(txn.id);

{
  intentMatch: true,
  policyCompliant: true,
  receipt: "captured",
  auditLog: "complete"
}
Verified • 99.8% Match Rate
